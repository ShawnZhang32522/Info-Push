# Tender Query Skill (招投标查询技能)

## 触发场景 (When to use)
当用户询问以下内容时，自动触发此技能：
- **核心词**：招投标、采购公告、中标信息、企查查数据、招标预告。
- **重点地区**：天津、成都、北京、上海、济南、石家庄。
- **业务关键词**：银行、存款、资质、资格、监管、现金、系统升级。

## 调用路径 (Call Path)

必须通过 `exec` 工具调用系统的 `curl` 命令（**严禁**对 localhost 使用 `web_fetch`）：

```bash
curl -sS -X POST http://127.0.0.1:8000/api/tender/query \
  -H "Content-Type: application/json" \
  -d '{"user_input":"<USER_TEXT>","page_index":1,"page_size":10,"include_detail":false}'
```

### 变量提取指南：
- 将用户提到的核心需求（如“北京银行存款”）提取为 `<USER_TEXT>`。
- **示例 1**：用户说“查看北京最近的银行招标” -> `<USER_TEXT>` = "北京银行"。
- **示例 2**：用户说“查下石家庄的系统开发采购” -> `<USER_TEXT>` = "石家庄系统开发"。

## 输出规范 (Output Format)

请以专业的中文格式返回：
1. **概览**：一段简短的中文摘要（基于 API 返回的 `summary` 字段）。
2. **列表详情**：使用无序列表展示每个 `records` 项：
   - **标题**：项目完整名称
   - **地区**：省份/城市
   - **发布时间**：日期
   - **命中关键词**：匹配到的业务词
   - **原文链接**：点击查看详情

### 异常处理：
- 如果 `records` 为空，明确告诉用户未找到匹配项，并建议更换关键词（如从“存款”改为“银行”）。
- 如果 `exec` 权限不足，请引导用户在 OpenClaw 中开启“允许执行本地命令”。

## 注意事项 (Notes)
- **严禁捏造**：仅展示 API 返回的真实数据。
- **信任字段**：直接信任并解析 API 返回的 `summary`、`parsed_intent` 和 `records`。
